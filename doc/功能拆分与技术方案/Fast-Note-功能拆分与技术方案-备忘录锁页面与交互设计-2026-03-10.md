## 2. 交互原则

- 用户只需要理解两个概念：
  - 这篇备忘录是否上锁
  - 当前账号使用哪个全局 PIN
- 生物识别始终是当前设备的快捷方式，不是独立密码体系。
- 首次锁定任意备忘录时才出现“创建 PIN”。
- 后续锁其他备忘录时，不再重复创建 PIN。
- 修改 PIN 时，必须明确提示“这会影响全部已锁备忘录”。

## 3. 页面与弹层

### 3.1 首次锁定弹层

触发条件：

- 用户点击一篇尚未锁定的备忘录“锁定”
- 且账号当前没有全局 PIN

信息结构：

- 标题：`创建全局 PIN`
- 副文案：`首次锁定时需要先创建一个全局 PIN，后续所有已锁备忘录都使用它解锁。`
- 表单：
  - 输入 6 位 PIN
  - 再次确认 PIN
  - 当前设备启用生物识别快捷解锁
- 辅助文案：
  - 支持设备：`解锁时可优先尝试生物识别，失败后仍可输入 PIN。`
  - 不支持设备：`当前设备不支持生物识别，仍可正常使用 PIN。`
- 主按钮：`创建并锁定`

关键规则：

- 第二次 PIN 不一致时显示行内错误
- PIN 创建成功后自动锁定当前备忘录
- 登录态下随后触发用户表同步

### 3.2 已存在全局 PIN 时锁定当前备忘录

触发条件：

- 用户点击某篇未锁定备忘录“锁定”
- 系统已存在全局 PIN

交互建议：

- 不再弹出创建 PIN 表单
- 直接执行锁定，并提示：
  - `已使用当前全局 PIN 锁定这篇备忘录`

可选增强：

- 若产品希望更强确认感，可展示轻量确认弹层，但不是必需

### 3.3 解锁面板

适用场景：

- 打开 `is_locked=1` 且无有效会话的备忘录

信息结构：

- 锁图标
- 标题：`这篇备忘录已锁定`
- 副文案：`输入全局 PIN 解锁，支持时也可以使用当前设备的生物识别。`
- 生物识别按钮：`尝试生物识别`
- PIN 输入区
- 错误提示区
- 提交按钮：`解锁`

关键规则：

- 生物识别按钮只在“已启用且设备支持”时展示
- 生物识别失败后停留当前面板，不跳页
- PIN 连续失败后进入冷却，显示倒计时
- PC 保持左侧列表不动，只锁住右侧详情区

### 3.4 锁管理弹层

触发条件：

- 用户点击已锁备忘录的“锁设置”

信息结构：

- 标题：`锁设置`
- 状态摘要：
  - `当前状态：这篇备忘录已受全局 PIN 保护`
  - `当前设备：已启用/未启用生物识别快捷解锁`
- 操作区：
  - `修改全局 PIN`
  - `立即重新锁定`
  - `关闭这篇备忘录的锁`
  - 生物识别开关

关键规则：

- `修改全局 PIN`：
  - 文案必须提示“会影响全部已锁备忘录”
- `立即重新锁定`：
  - 只影响当前备忘录
- `关闭这篇备忘录的锁`：
  - 只取消当前 note 的 `is_locked`
  - 不删除全局 PIN

## 4. 文案建议

### 4.1 首次创建 PIN

- 标题：`创建全局 PIN`
- 副文案：`首次锁定时创建一次，之后所有已锁备忘录共用这个 PIN。`

### 4.2 锁定成功

- `已锁定这篇备忘录`
- `已使用当前全局 PIN 锁定这篇备忘录`

### 4.3 修改 PIN

- 按钮：`修改全局 PIN`
- 提示：`修改后，全部已锁备忘录都将使用新 PIN 解锁。`
- 成功：`全局 PIN 已更新`

### 4.4 错误提示

- `请输入 6 位数字 PIN`
- `两次输入的 PIN 不一致`
- `PIN 不正确，请重试`
- `PIN 输入错误次数过多，请稍后再试`
- `当前设备不支持生物识别，请输入 PIN 解锁`

## 5. PC / 移动端布局建议

### 5.1 PC

- 解锁面板放在详情页右栏中心，不遮挡左侧上下文
- 锁设置与首次创建 PIN 采用居中 modal，宽度 420 到 460px
- 管理弹层信息层次要清楚地区分：
  - 当前 note 操作
  - 全局 PIN 操作

### 5.2 移动端

- 首次创建 PIN 和锁管理采用底部 sheet
- 解锁面板占据详情主内容区，按钮与输入区域保持单列
- 冷却和错误提示放在输入框下方，避免 toast 一闪而过

## 6. 状态机

| 状态 | 含义 | UI | 可执行动作 | 结束条件 |
| ---- | ---- | -- | ---------- | -------- |
| `unlocked` | note 未锁或当前会话有效 | 编辑器可见 | 编辑、管理锁 | 手动重锁、会话过期 |
| `locked` | note 已锁且无会话 | 显示解锁面板 | 输入 PIN、尝试生物识别 | PIN 成功 / 生物识别成功 |
| `unlocking` | 正在校验 | 按钮 loading、输入禁用 | 等待结果 | 成功 / 失败 |
| `cooldown` | PIN 连续失败过多 | 显示倒计时 | 等待 | 倒计时结束 |

## 7. 关键流程图

### 7.1 首次锁定

```text
点击“锁定”
  -> 系统无全局 PIN
  -> 打开“创建全局 PIN”
  -> 输入 PIN / 确认 PIN
  -> 可选开启当前设备生物识别
  -> 创建成功
  -> 当前 note 锁定
```

### 7.2 后续锁定

```text
点击“锁定”
  -> 系统已有全局 PIN
  -> 直接锁定当前 note
  -> 给成功提示
```

### 7.3 解锁

```text
打开已锁 note
  -> 显示解锁面板
  -> 尝试生物识别
    -> 成功：进入编辑器
    -> 失败：继续停留
  -> 输入全局 PIN
    -> 正确：进入编辑器
    -> 错误过多：进入冷却
```

## 8. 可用性要求

- PIN 输入区是首要焦点
- 生物识别入口是增强能力，不可挤掉 PIN 主路径
- 修改全局 PIN 时必须强化影响范围提示
- 所有成功/失败文案都要区分“全局 PIN”和“当前备忘录锁”，避免概念混淆

## 9. 本轮口径结论

- 不再设计“每篇备忘录不同密码”
- 不再设计 legacy 迁移页
- 首次锁定时创建全局 PIN
- 后续锁定仅控制当前 note 的 `is_locked`
- 全局 PIN 同步到 PocketBase `users` 表
