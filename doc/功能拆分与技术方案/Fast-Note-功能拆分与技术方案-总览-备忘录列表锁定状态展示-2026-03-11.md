## 4. 里程碑与发布建议

- 里程碑划分：
  - M1：在 `NoteListItem` 为 note 行增加左侧状态槽位，接入 `is_locked` 展示逻辑
  - M2：补主题色和尺寸样式，确保浅色/深色模式下视觉一致
  - M3：补最小测试或冒烟清单，覆盖锁定、未锁定、长标题、多列表入口
- 上线策略：
  - 可直接随常规前端迭代上线，不需要数据迁移或灰度分表。
- 灰度/回滚：
  - 若出现标题错位或主题可读性问题，可只回退 `NoteListItem` 的锁图标插槽和样式，不影响锁定能力本身。

## 5. 跨模块依赖与风险

- 依赖关系：
  - 依赖 `Note.is_locked` 已在类型层定义并在 store 初始化阶段标准化
  - 依赖 `NoteList` 全量复用 `NoteListItem` 渲染文件夹与 note 行
  - 依赖现有主题切换机制通过 `ion-palette-dark` 切换 token
- 关键风险：
  - 若直接在 note 行插入图标而不保留未锁定占位，列表标题会左右抖动。
  - 若图标颜色直接写死，深色模式下可能与背景对比不足。
  - 若占位宽度与文件夹图标区域差异过大，会破坏现有列表扫描节奏。
- 需要业务决策的问题：
  - 锁图标颜色是否直接复用现有图标 token `--c-icon`，还是采用 `primary` 同级强调色。
  - 多行标题时锁图标采用文字基线对齐还是视觉居中对齐。
