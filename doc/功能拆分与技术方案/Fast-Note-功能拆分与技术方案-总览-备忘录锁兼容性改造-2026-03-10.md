- 输出范围：备忘录锁改为“全局唯一 PIN + 按 note 开关 `is_locked` + 当前设备生物识别快捷解锁 + 全局 PIN 同步到 PocketBase `users` 表”

## 2. 版本结论

- 当前方案不再采用“每篇备忘录独立 PIN”。
- 用户账号只维护 1 个全局 PIN。
- 备忘录是否受保护，继续由 note 上的 `is_locked` 控制。
- 用户首次锁定任意备忘录时，如果尚未存在全局 PIN，则先创建 PIN，再锁定当前备忘录。
- 全局 PIN 需要同步到 PocketBase `users` 默认表。
- 生物识别仍然只属于当前设备能力，不上云。
- 当前开发板不再考虑 legacy WebAuthn 迁移。

| 模块 | 功能点 | 用户价值 | 优先级 | 当前状态 | 备注 |

| 备忘录锁全局 PIN 改造 | F1.1 首次锁定创建全局 PIN | 用户只记一个 PIN，后续锁更多备忘录无心智负担 | P0 | 需重构 | 当前实现仍是每 note 一个 PIN |

## 4. 数据与同步策略

### 4.1 本地

- `notes`：
  - 保留 `is_locked`
  - 不再承载 PIN hash/salt
- 新增 `security_settings` 本地表：
  - `scope_key`
  - `pin_secret_salt`
  - `pin_secret_hash`
  - `pin_version`
  - `updated`
- 保留 `device_security_state`：
  - 只存生物识别凭据和当前设备开关
- 保留 `note_unlock_sessions`：
  - 继续按 `note_id` 记录本地解锁会话、失败次数和冷却

### 4.2 云端

- PocketBase `users` 表新增字段：
  - `note_lock_pin_salt`
  - `note_lock_pin_hash`
  - `note_lock_pin_version`
- 同步原则：
  - note 的锁开关仍随 `notes` 正常同步，只同步 `is_locked`
  - 全局 PIN 不进入 `notes`
  - 全局 PIN 由用户资料同步链路读写 `users`
  - 生物识别不进入云端

## 5. 主流程摘要

### 5.1 首次锁定任意备忘录

1. 用户点击“锁定”
2. 系统检查本地/服务端是否已有全局 PIN
3. 若没有：
   - 弹出“创建全局 PIN”表单
   - 用户输入并确认 6 位 PIN
   - 本地写入 `security_settings`
   - 登录态下同步到 `users` 表
4. 将当前 note 的 `is_locked` 置为 `1`

### 5.2 锁定后打开备忘录

1. 检查 note 的 `is_locked`
2. 若为 `1`：
   - 读取当前 note 的本地会话
   - 会话有效则直接进入
   - 会话无效则展示解锁面板
3. 解锁面板优先提供：
   - 生物识别快捷入口
   - 全局 PIN 输入入口

### 5.3 管理已锁备忘录

- 关闭当前备忘录锁：
  - 仅将当前 note 的 `is_locked` 置为 `0`
  - 不删除全局 PIN
- 修改 PIN：
  - 修改的是账号级全局 PIN
  - 已锁备忘录后续全部使用新 PIN
- 立即重新锁定：
  - 仅清除当前 note 会话

## 6. 实施建议

- M1：收口数据模型，去掉 note 上的 PIN 字段，引入 `security_settings`
- M2：重构 `useNoteLock` 为“全局 PIN 校验 + 按 note 会话”
- M3：接入 `users` 表同步与账号切换场景
- M4：补自动化和文案

## 7. 关键风险

| 风险 | 说明 | 应对 |
| ---- | ---- | ---- |
| 用户误解为内容加密 | 本期仍是门禁，不是端到端加密 | 在文案和设置页明确说明 |
| `users` 表字段同步失败 | 会导致多设备 PIN 不一致 | 保留本地副本，增加拉取覆盖与错误提示 |
| 当前中间态代码与新文档不一致 | 之前已实现的是每 note 一个 PIN | 后续开发按本文件重构，不再沿旧模型推进 |
| 游客态与登录态切换 | 游客本地 PIN 无法天然落到账号 | 本期作为边界场景单独记录并测试 |

## 8. 本轮决策结果

- 决策日期：2026-03-10
- 决策结论：
  - 只保留 1 个全局 PIN
  - 首次锁定时创建 PIN
  - PIN 同步到 PocketBase `users` 表
  - 不继续推进 legacy 迁移
