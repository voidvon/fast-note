## 4. 里程碑与发布建议

- 里程碑划分：
  - M1（P0 稳定性）：修复公开页参数错配、补齐删除递归策略、补同步失败补偿文档
  - M2（P1 体验）：完善搜索、导航恢复边界、手动同步可观测性
  - M3（P1 安全）：WebAuthn 多设备策略与异常提示优化
- 上线策略：先灰度内部用户（同步与公开路径），再全量
- 灰度/回滚：
  - 同步异常时关闭自动同步入口，仅保留本地编辑
  - 公开路由异常时回退到只保留 `/:username` 首页入口

## 5. 跨模块依赖与风险

- 依赖关系：
  - `认证` -> `同步与实时`（登录态是同步前置）
  - `笔记编辑` -> `附件`（内容结构内嵌 file-upload）
  - `路由导航` -> `公开访问`（用户名路由触发公开数据初始化）
- 关键风险：
  - 公开页桌面模式传参 `note-uuid` 与详情页 `noteId` 定义不一致，可能导致详情不加载（`src/views/UserPublicNotesPage.vue:168`, `src/views/NoteDetail.vue:23`）
  - 文件夹删除仅处理一层子项，深层后代软删一致性需补强（`src/components/LongPressMenu.vue:70`）
  - `useDeviceType` 全局永久监听 `resize`，长期运行下可能产生资源开销（`src/hooks/useDeviceType.ts:15`）
- 需要业务决策的问题：
  - 扩展系统入口是否下线（代码已标注废弃，`src/hooks/useExtensions.ts:18`）
  - 公开分享权限是否需要“范围级别”而非二值开关

### 8.4 跨模块依赖与风险增补

- 依赖关系：
  - `同步与实时` -> `路由与导航`：私有详情路由恢复时机需要与现有桌面刷新恢复逻辑对齐。
  - `同步与实时` -> `笔记编辑与组织`：`NoteDetail` 新建/更新判定需要保持移动端新建链路兼容。

- 关键风险：

- 需要业务决策的问题：
  - 本地缓存补齐是自动触发还是需要用户确认。
  - 切账号后是否提供“清理旧账号本地缓存”入口。
