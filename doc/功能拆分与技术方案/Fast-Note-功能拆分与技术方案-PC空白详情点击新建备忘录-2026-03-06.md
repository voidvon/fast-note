# PC 空白详情点击新建备忘录技术方案（Fast-Note）

## 1. 文档信息

- 功能名称：PC 空白详情区点击启动新建备忘录
- 输出日期：2026-03-06
- 输出文件：`doc/功能拆分与技术方案/Fast-Note-功能拆分与技术方案-PC空白详情点击新建备忘录-2026-03-06.md`
- 需求来源：用户新增需求“当备忘录没有被选中时，此时右侧的详情是空的，点击空白处可以启动新建备忘录的编辑器，类似点击中间栏的新建备忘录按钮，并聚焦到富文本中”
- 关联模块：笔记编辑与组织、首页桌面分栏
- 不在范围：移动端新建交互、标题默认值策略调整、编辑器工具栏改版、保存/同步策略改造

## 2. 目标与现状

- 业务目标：在桌面端首页右侧详情为空时，提供直接的新建入口，让用户能从空白区直接开始记录。
- 当前状态：部分支持。
- 代码证据：
  - `E1` `src/views/HomePage.vue:224`：桌面端切换目录时会把 `state.noteId` 重置为空字符串，右侧进入“无选中笔记”状态。
  - `E2` `src/views/HomePage.vue:262`：桌面端仅在 `allnotes/deleted` 等默认目录场景下自动兜底首条笔记，非默认场景仍可能出现右侧空白态。
  - `E3` `src/views/HomePage.vue:279`：现有 `handleCreateNote(parentId)` 已封装桌面端新建入口，核心行为是设置 `parentId` 并将 `noteId` 置为 `'0'`。
  - `E4` `src/views/HomePage.vue:371`：桌面端底部“新建备忘录”按钮已经直接复用 `handleCreateNote()`。
  - `E5` `src/views/HomePage.vue:414`：右侧详情区当前始终渲染 `NoteDetail`，没有专门的空态提示或点击创建入口。
  - `E6` `src/views/NoteDetail.vue:98`：当 `noteId === '0'` 时，编辑器会立即清空内容并进入新建态。
  - `E7` `src/views/NoteDetail.vue:110`：进入新建态后，`NoteDetail` 已有延迟聚焦富文本正文的逻辑。
  - `E8` `src/views/NoteDetail.vue:117`：当 `noteId` 为空时，详情区仅清空编辑器，不会自动触发创建。
- 现状结论：当前“创建并聚焦正文”的能力已经存在，但只挂在底部按钮和中栏入口上；右侧空白详情区缺少复用该能力的点击入口。

## 3. 设计原则与推荐方案

### 3.1 设计原则

- 复用现有新建链路：避免新增第二套创建逻辑，保持 `handleCreateNote()` 与 `NoteDetail` 新建态为唯一入口。
- 保持 offline-first：不修改 `Note` 契约、Dexie schema、同步链路，创建仍以本地 store 为即时真相源。
- 保持桌面语义清晰：仅在桌面端右侧详情区的空态生效，不影响移动端和已有详情编辑态。
- 保持聚焦一致：进入新建态后继续使用 `NoteDetail` 现有正文聚焦逻辑，不在 `HomePage` 重复实现焦点控制。

### 3.2 方案对比

#### 方案 A：在 `HomePage` 为空态时渲染全区域可点击占位层（推荐）

- 思路：桌面端当 `state.noteId` 为空时，在右侧详情区渲染空态占位层，点击后直接调用 `handleCreateNote()`。
- 优点：
  - 改动集中在 `HomePage` 表现层；
  - 可直接复用既有新建链路；
  - 用户可见性更强，后续也方便补提示文案/图标。
- 缺点：
  - 需要补一层空态样式和点击反馈；
  - 需避免在 `noteId='0'` 的新建态重复触发。
- 结论：推荐实施。

#### 方案 B：由 `NoteDetail` 内部识别空态并拦截点击创建

- 思路：继续始终渲染 `NoteDetail`，由其在 `noteId` 为空时显示点击创建态，并通过事件回传给 `HomePage`。
- 优点：逻辑都在详情区组件内，视图看起来更“内聚”。
- 缺点：
  - `NoteDetail` 需要感知“空态创建”这种页面编排语义；
  - 新建所需的 `parentId`、桌面分栏上下文仍在 `HomePage`，会产生额外跨层事件。
- 结论：本次不采用。

## 4. 详细技术方案

### 4.1 触发条件

- 生效范围：桌面端首页右侧详情区。
- 生效条件：
  - `isDesktop === true`
  - 当前 `state.noteId === ''`
- 不重复触发条件：
  - 当 `state.noteId === '0'`（已进入新建态）时，不再显示空态点击层。
  - 当 `state.noteId` 为真实笔记 ID 时，继续展示正常详情，不响应空态创建。

### 4.2 页面结构调整

- `src/views/HomePage.vue`
  - 在 `.home-detail` 容器内增加显式空态渲染分支。
  - 推荐结构：
    1. `state.noteId` 为空时，渲染一个占满右侧区域的空态容器；
    2. 点击空态容器后调用 `handleCreateNote()`；
    3. `state.noteId` 变为 `'0'` 或真实 ID 后，切换回 `NoteDetail`。
- 空态容器建议包含：
  - 主提示：`点击开始新建备忘录`
  - 辅助提示：`将直接进入编辑并聚焦正文`
  - 轻量 hover/cursor 反馈

### 4.3 创建链路复用

- 点击空态后直接复用 `handleCreateNote()`。
- `handleCreateNote()` 继续承担：
  - 设定 `state.parentId`
  - 设定 `state.noteId = '0'`
  - 清理桌面快照（若已接入快照逻辑）
- `NoteDetail` 继续承担：
  - 识别 `'0'` 新建态
  - 初始化新笔记 ID
  - 清空编辑器内容
  - 聚焦富文本正文

### 4.4 聚焦与防重复策略

- 焦点策略：
  - 不在 `HomePage` 新增焦点逻辑；
  - 继续沿用 `NoteDetail` 当前 `noteId==='0'` 时的 `nextTick + setTimeout + focus()` 链路。
- 防重复策略：
  - 空态层仅在 `state.noteId === ''` 时显示；
  - 一旦进入新建态（`'0'`），空态层立即消失，避免连续点击创建多个新建会话。

### 4.5 边界与降级

- 当前目录为空文件夹：允许点击空白详情区进入新建态，创建结果与底部“新建备忘录”按钮一致。
- 当前处于回收站列表：默认沿用现有底部按钮行为，即仍可触发 `handleCreateNote()`；是否限制为不可创建本期不调整。
- 若 `NoteDetail` 初始化延迟导致首次未聚焦：保持现有延迟聚焦机制；必要时在实施阶段补一轮焦点回归验证。

## 6. 关键实现步骤

### 6.1 空态显式化

- 在 `HomePage` 中新增计算条件，如 `isEmptyDesktopDetail`：
  - `isDesktop && !state.noteId`
- 在 `.home-detail` 中按条件切换：
  - 空态：渲染点击创建层；
  - 非空态：渲染 `NoteDetail`。

### 6.2 点击复用现有新建行为

- 将空态层点击事件绑定到 `handleCreateNote()`。
- 不新增新的 `createDraftFromBlankDetail()` 等并行函数，避免逻辑分叉。

### 6.3 聚焦回归验证

- 验证从空态点击进入新建后：
  - 编辑器已加载；
  - 富文本正文自动聚焦；
  - 用户可直接输入正文。
- 如发现挂载时序问题，再在 `NoteDetail` 内做最小补丁，禁止在 `HomePage` 手动操作编辑器实例。

## 9. 验收标准

- AC-001：桌面端在右侧无选中笔记时，点击空白详情区可进入新建编辑器。
- AC-002：通过右侧空白区点击进入的新建流程，与底部“新建备忘录”按钮行为一致。
- AC-003：进入新建编辑器后，富文本正文自动获得焦点，用户可直接输入。
- AC-004：当右侧已经展示真实笔记详情时，点击详情内容区不会误创建新笔记。
- AC-005：连续点击空白详情区不会产生多个并发新建会话。
- AC-006：移动端新建与详情展示行为不受本次改动影响。

## 11. 风险与待确认

### 11.1 风险

- R1：若空态容器与 `NoteDetail` 的占位 DOM 重叠处理不当，可能出现点击穿透或 hover 区域异常。
- R2：若 `NoteDetail` 新建态挂载时序存在波动，可能出现从空态点击进入后焦点偶发失败。
- R3：当前回收站场景底部按钮已允许创建，新入口也会继承该行为；若产品后续不接受，需要单独补规则。

### 11.2 待确认

- Q1：空态是否需要固定提示文案/图标，还是仅保留“整块可点”的轻量交互。
- Q2：在回收站列表下的空态点击是否继续允许创建；当前方案默认与现有底部按钮一致。
- Q3：是否需要为空态点击创建增加埋点或日志；当前方案默认不增加。

## 12. 结论

- 推荐采用“`HomePage` 空态显式可点击 + 复用现有 `handleCreateNote()` + 复用 `NoteDetail` 聚焦逻辑”的最小方案。
- 该方案只动表现层与少量详情聚焦回归，不改变领域模型、数据库、同步、路由语义，风险可控。
- 任务粒度清晰，适合作为 P1 体验优化项进入当前开发计划。
