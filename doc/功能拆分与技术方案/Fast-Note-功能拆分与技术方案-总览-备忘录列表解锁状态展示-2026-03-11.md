## 4. 里程碑与发布建议

- 里程碑状态：
  - M1：列表图标从“静态 `is_locked`”升级为“`is_locked` + 本地解锁会话”，已完成。
  - M2：补齐解锁成功、手动重新锁定、会话自然过期三类刷新机制，已完成。
  - M3：补单测、集成测试与最小冒烟建议，已完成。
- 上线策略：
  - 属于现有列表锁图标能力的增量增强，不涉及 schema 或云端协议变更。
- 回滚边界：
  - 若后续发现列表状态错乱，可单独回退 `features/note-lock` 的列表展示态派生逻辑，而不影响 note 锁主流程。

## 5. 跨模块依赖与风险

- 依赖关系：
  - 依赖 `src/features/note-lock/model/use-note-lock.ts` 已有的本地会话判定与事件派发能力。
  - 依赖 `src/widgets/note-list/ui/note-list.vue` 作为首页、文件夹页等列表入口的统一渲染组件。
  - 依赖 `src/features/note-lock/ui/note-lock-manage-modal.vue` 正确接入 `relock()`。
- 当前风险：
  - 暂无 e2e 覆盖真实页面切换和长时间停留链路。
  - 当前集成测试主覆盖首页列表，文件夹页依赖同一 widget 共享实现，残余风险较低但仍存在。
- 已明确保持的不变量：
  - `Note` 契约不新增字段。
  - 解锁态仍是设备本地会话语义，不参与云端同步。
  - `cooldown` 不会映射成解锁态。
