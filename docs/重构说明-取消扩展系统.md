# PocketBase 扩展系统迁移说明

## 文档定位

本文记录 PocketBase 能力从早期扩展系统逐步迁入当前 FSD 架构的过程。扩展目录本身已经退出主链路，当前真实落点如下。

## 当前落点

- 认证：`fastnote/src/processes/session` + `fastnote/src/shared/api/pocketbase/auth.ts`
- Realtime：`fastnote/src/processes/session` + `fastnote/src/shared/api/pocketbase/realtime-service.ts`
- 同步：`fastnote/src/processes/sync-notes`
- 公开笔记：`fastnote/src/processes/public-notes`
- PocketBase API：`fastnote/src/shared/api/pocketbase/*`
- PocketBase 相关类型：`fastnote/src/shared/types/pocketbase.ts`
- 通用认证契约：`fastnote/src/shared/lib/auth/*`
- 通用 Realtime 契约：`fastnote/src/shared/lib/realtime/*`

## 历史路径到当前路径

| 历史路径 | 当前路径 |
| -------- | -------- |
| `extensions/pocketbase/api/client` | `fastnote/src/shared/api/pocketbase/*` |
| `extensions/pocketbase/hooks/useAuth` | `fastnote/src/processes/session` |
| `extensions/pocketbase/hooks/useSync` | `fastnote/src/processes/sync-notes` |
| `extensions/pocketbase/hooks/useRealtime` | `fastnote/src/processes/session` |
| `extensions/pocketbase/types` | `fastnote/src/shared/types/pocketbase.ts` |

## 当前推荐用法

### 认证

```ts
import { useAuth } from '@/processes/session'

const { login, logout, currentUser } = useAuth()
```

### 同步

```ts
import { useSync } from '@/processes/sync-notes'

const { sync, syncing, syncStatus } = useSync()
```

### Realtime

```ts
import { useRealtime } from '@/processes/session'

const { connect, disconnect, isConnected } = useRealtime()
```

### 类型

```ts
import type { PublicUserInfo } from '@/shared/types/pocketbase'
```

## 架构收益

1. PocketBase 不再以“可选扩展”存在，而是作为 `shared` 基础设施与 `processes` 会话流程的组成部分。
2. 页面和 widget 不再直接依赖扩展目录或历史适配器目录。
3. 会话初始化、同步、公开访问都能在统一流程层管理。

## 当前结论

这轮迁移已经从“取消扩展系统”进一步推进到“旧兼容入口下线”。后续新增代码不应再引入 `hooks/core/adapters/services` 这套中间时期目录概念，而应直接落到当前 FSD 目录。
