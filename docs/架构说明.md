# 项目架构说明

## 当前状态

fastnote 当前采用前后端一体仓库结构：

- `fastnote/`：前端应用，按 FSD 分层
- `backend/`：PocketBase Go 宿主
- `docs/`：开发、产品与架构文档

当前后端仅承担宿主职责：

- 启动 PocketBase
- 预留 hooks 与 migrations 目录
- 挂载前端静态资源

当前不包含业务 routes、业务表定义和业务 migrations。

## 前端职责

### `fastnote/src/app`

- 应用启动
- 路由装配
- 全局 provider
- 应用级初始化

### `fastnote/src/processes`

- 会话初始化
- 同步编排
- 公开笔记初始化
- 导航恢复

### `fastnote/src/pages`

- 路由页装配
- 页面级参数适配

### `fastnote/src/widgets`

- 业务组合 UI

### `fastnote/src/features`

- 用户动作与用例

### `fastnote/src/entities`

- 实体状态
- 规则封装
- 查询与组合

### `fastnote/src/shared`

- Dexie、本地存储
- PocketBase SDK 适配
- 通用工具、类型、UI

## 后端职责

### `backend/main.go`

- 创建并启动 PocketBase
- 注册 bootstrap
- 注册 hooks
- 注册 migrations

### `backend/internal/server/bootstrap`

- 挂载 `fastnote/dist`
- 为一体化部署提供静态资源宿主能力

### `backend/internal/server/hooks`

- 预留服务端 hook 目录
- 当前不承载业务实现

### `backend/migrations`

- 预留 PocketBase schema 与设置变更目录
- 当前不承载业务迁移

## 依赖方向

### 前端

- `app -> processes -> pages/widgets/features -> entities -> shared`

补充规则：

- `pages` 可以依赖 `widgets`、`features`、`entities`、`shared`
- `widgets` 可以依赖 `features`、`entities`、`shared`
- `features` 可以依赖 `entities`、`shared`
- `entities` 只能依赖 `shared`
- `shared` 不反向依赖业务层

### 前后端

- `fastnote/` 通过 PocketBase SDK、realtime 和文件接口访问 `backend/`
- `backend/` 不依赖 `fastnote/src` 业务代码

## 当前约束

1. 前端继续保持 offline-first。
2. 前端默认走同源或显式配置的 PocketBase 地址。
3. 后端当前只做宿主，不主动增加业务代码。
4. 正式的服务端 schema 或规则变更应进入 `backend/migrations` 与 `backend/internal/server/hooks`。
