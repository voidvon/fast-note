# 项目架构说明

## 当前状态

Fast-Note 当前采用 FSD + DDD 分层，代码主路径已经统一到：

- `app`：启动、路由、全局装配
- `processes`：跨页面长流程，如会话、导航、同步、公开笔记初始化
- `pages`：路由页容器
- `widgets`：业务组合 UI
- `features`：用户动作与用例
- `entities`：领域模型与状态
- `shared`：基础设施、通用工具、公共契约

`components/core/utils/types/database/pocketbase/adapters` 等历史兼容目录已从代码层删除，不再作为开发入口。

## 核心职责

### 会话与认证

- `src/processes/session` 负责登录态初始化、游客态数据决策、Realtime 建连、同步触发与私有路由恢复。
- `src/shared/api/pocketbase` 提供认证、用户、笔记、文件等远端 API。
- `src/shared/lib/auth` 与 `src/shared/lib/realtime` 提供通用契约。

应用启动入口当前由 [App.vue](/root/fast-note/src/App.vue) 触发：

```ts
import { onMounted, onUnmounted } from 'vue'
import { useTheme } from '@/features/theme-switch'
import { useSessionBootstrap } from '@/processes/session'

const { initTheme } = useTheme()
const { initializeSession, disposeSession } = useSessionBootstrap()

onMounted(async () => {
  initTheme()
  await initializeSession()
})

onUnmounted(() => {
  disposeSession()
})
```

### 路由与公开访问

- `src/app/router` 负责路由表和全局守卫。
- `src/processes/navigation` 负责导航历史、桌面活跃笔记、路由恢复、智能返回。
- `src/processes/public-notes` 负责公开笔记路由就绪、公开用户缓存、公开数据同步。

当前路由入口由 [src/app/router/index.ts](/root/fast-note/src/app/router/index.ts) 装配：

```ts
import { createRouter, createWebHistory } from '@ionic/vue-router'
import { registerRouterDependencies } from '@/processes/navigation/model/register-router-dependencies'
import { ensurePublicNotesRouteReady, shouldRedirectDesktopNoteRoute } from '@/processes/public-notes/model/ensure-public-notes-route-ready'
import { appRoutes } from './routes'
```

### 笔记领域

- `src/entities/note` 承载笔记、文件夹树、删除保留期、父子关系等领域规则。
- `src/entities/public-note` 承载公开只读投影与用户名隔离的数据集。
- `src/features/note-*` 负责保存、移动、删除、锁定、详情页编排等动作。
- `src/widgets/note-list`、`src/widgets/editor` 等承载业务组合 UI。

## 依赖方向

必须保持单向依赖：

- `app` -> `processes` -> `pages`/`widgets`/`features` -> `entities` -> `shared`

补充规则：

- `pages` 可以依赖 `widgets`、`features`、`entities`、`shared`
- `widgets` 可以依赖 `features`、`entities`、`shared`
- `features` 可以依赖 `entities`、`shared`
- `entities` 只能依赖 `shared`
- `shared` 不能反向依赖业务层

## 迁移建议

如果要继续推进迁移，优先顺序建议为：

1. 更新技能参考与开发文档，避免再生成历史路径。
2. 清理空的历史目录壳与样式遗留目录。
3. 继续把跨上下文规则从页面和 widget 往 `features/entities/processes` 下沉。

## 总结

当前架构的重点不再是“把扩展并入核心”，而是持续减少历史痕迹，保持新增逻辑只落在 `app/processes/pages/widgets/features/entities/shared` 这套路径里。
