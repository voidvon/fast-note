  - `部分支持`：附件上传过程仍缺少用户可见进度和失败重试 UI。
  - `部分支持`：附件引用提取与回填仍依赖 HTML 正则，结构鲁棒性一般。
  - `部分支持`：锁模型已升级为全局 PIN，但部分 UI 文案仍保留“备忘录密码”口径。

## 3. 当前实现拆解

### 3.1 F5.1 附件本地 hash 管理与去重上传

#### 3.1.1 目标与边界

- 用户场景：用户在编辑器中插入图片或文件，同一文件重复引用时不重复落库、不重复上传。
- 范围内：hash 计算、本地 `note_files` 存储、富文本 file-upload 节点、同步映射、上传回填。
- 范围外：缩略图、断点续传、服务端文件治理。

#### 3.1.2 当前实现

- `shared/lib/file-hash`
  - 已提供文件 hash 计算能力。
- `entities/note`
  - `useNoteFiles()` 负责把原始文件和元信息写入 `note_files` 表，并按 hash 读取。
- `features/note-editor`
  - 插入文件时先算 hash，再检查本地是否已存在。
  - 若本地不存在则写入 Dexie；富文本节点的 `url` 先写 hash。
  - 当前 `extractFileHashes()` 仍通过正则从 HTML 中提取 `file-upload` 引用。
- `processes/sync-notes`
  - 同步前会把 hash 引用映射成临时占位符和上传文件列表。
  - 同一 hash 在单条笔记内会复用同一个临时占位符，不重复上传。
  - 服务端返回真实文件名后，会再次回写内容和 `files` 列表。
- `shared/api/pocketbase`
  - 支持 `FormData` 上传文件，并在远端回读时按 `noteId + filename` 解析访问 URL。

#### 3.1.3 迁移状态

- 状态：部分支持
- 当前结论：
  - 附件链路已经迁到 `shared/lib/file-hash + entities/note + features/note-editor + processes/sync-notes`。
  - 当前缺口集中在结构鲁棒性和可观测性，不是“附件还没法上传”。

#### 3.1.4 剩余风险与建议

- 风险：附件引用提取和回填仍依赖 HTML 正则匹配，后续编辑器序列化结构变化时存在脆弱性。
- 风险：附件上传过程目前没有用户可见的进度和重试 UI。
- 建议：后续优先把附件引用解析升级为编辑器结构遍历或 DOM 解析，再补上传状态反馈。

#### 3.1.5 验证建议

- 冒烟：插入相同文件两次 -> 保存 -> 同步 -> 跨端打开，确认仅上传一次且可读。
- 回归关注点：已是远端文件名的引用不能被错误当成 hash 再次上传。

### 3.2 F5.2 全局 PIN 与生物识别快捷解锁

#### 3.2.1 目标与边界

- 用户场景：用户可为敏感笔记启用锁，访问时需通过全局 PIN 解锁，并可在当前设备启用生物识别快捷解锁。
- 范围内：全局 PIN 创建/修改、设备级生物识别开关、锁定开关、解锁会话、失败冷却、锁定视图。
- 范围外：服务端零知识方案、跨设备同步生物识别凭据。

#### 3.2.2 当前实现

- `features/note-lock`
  - 会在登录态下把云端 PIN 配置同步回本地安全设置。
  - 启用笔记锁前会先确认是否已有全局 PIN。
  - 首次锁定时支持先创建全局 PIN，再立即为当前笔记启锁。
  - 支持修改全局 PIN，并在登录态下同步到云端用户配置。
  - PIN 校验失败会累计次数，达到阈值后进入冷却窗口。
  - 解锁成功会创建本地会话，短时间内可复用。
  - 生物识别只在当前设备有效，作为 PIN 的快捷解锁入口。
- `pages/note-detail`
  - 私有笔记加载后会先进入锁视图流转，而不是直接显示正文。
- `widgets/note-more`
  - 已接入锁设置和管理入口。
- `features/note-lock/ui`
  - 解锁面板已支持生物识别入口、PIN 输入、失败计数和冷却提示。
  - 但当前文案仍把全局 PIN 描述成“备忘录密码”。

#### 3.2.3 迁移状态

- 状态：部分支持
- 当前结论：
  - 主安全模型已经不是旧的单 note 密码，而是“云端同步全局 PIN + 本地设备生物识别快捷解锁 + note 会话”。
  - 当前缺口主要在文案、设备说明和自动化覆盖，不在主流程本身。

#### 3.2.4 剩余风险与建议

- 风险：生物识别凭据仍是设备本地能力，不随账号跨设备同步，产品文案需要持续强调这一点。
- 风险：云端 PIN 同步失败时，不同设备之间可能短时间出现配置漂移。
- 风险：当前解锁面板文案还没有完全切到“全局 PIN”口径。
- 建议：后续优先补文案统一和云端 PIN / 本地生物识别边界说明，再补更完整的自动化验证。

#### 3.2.5 验证建议

- 冒烟：创建全局 PIN -> 启用锁 -> 关闭并重新打开笔记 -> PIN/生物识别解锁 -> 重新锁定。
- 回归关注点：公开上下文和未锁定笔记不能误进入锁视图；生物识别不可用时必须稳定回退到 PIN。
