## 4. 里程碑与发布建议

- 里程碑状态：
  - M1：补编辑器默认块能力与详情页新建场景接入，已完成。
  - M2：补语义空内容判定，阻断默认空标题误保存，已完成。
  - M3：补单元/集成测试与最小冒烟建议，已完成。
- 上线策略：
  - 属于编辑器初始化和保存判定优化，不涉及数据迁移。
- 回滚边界：
  - 若后续出现误保存或格式异常，可单独回退默认标题初始化链路，而保留语义空内容判定。

## 5. 跨模块依赖与风险

- 依赖关系：
  - 依赖 `src/features/note-detail-editor/model/use-note-detail-editor-state.ts` 的新建草稿入口。
  - 依赖 `src/widgets/editor/ui/yy-editor.vue` 对页面暴露默认标题和语义判定能力。
  - 依赖 `src/features/note-save/model/use-note-save.ts` 保持 offline-first 本地优先保存语义。
- 当前风险：
  - 当前测试主要基于现有页面 helper 和 expose API stub，没有专门的真实 Tiptap 实例级 e2e。
- 已明确保持的不变量：
  - `Note` 仍只以 `content: string` 承载正文。
  - 已有笔记不会被自动改写为一级标题首块。
  - 语义空内容判定不会把附件、表格、任务列表误判成空内容。
