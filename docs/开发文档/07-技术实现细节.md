# 技术实现细节

本文档描述当前项目的关键实现策略，按当前前后端一体结构整理。

## 1. 响应式布局

前端根据屏幕宽度在桌面与移动之间切换布局：

- 桌面端使用分栏结构
- 移动端使用列表页与详情页切换

相关实现主要位于：

- `fastnote/src/pages/*`
- `fastnote/src/processes/navigation/*`
- `fastnote/src/shared/lib/device/*`
- `fastnote/src/shared/lib/viewport/*`

## 2. 本地优先存储

项目继续采用 offline-first：

1. 用户操作先落本地状态
2. 本地状态持久化到 IndexedDB
3. 同步流程再将增量写入云端
4. 页面刷新或离线场景优先从本地恢复

相关实现主要位于：

- `fastnote/src/shared/lib/storage/*`
- `fastnote/src/entities/note/model/state/*`

## 3. PocketBase 访问方式

前端通过 PocketBase JavaScript SDK 访问后端。

当前地址策略：

1. 优先使用 `VITE_POCKETBASE_URL`
2. 未配置时在浏览器环境使用当前 `origin`
3. 最后回退到 `http://127.0.0.1:8090`

相关实现：

- [client.ts](/Users/yytest/Documents/projects/fast-note/fastnote/src/shared/api/pocketbase/client.ts)

## 4. 同步策略

同步流程由前端 `processes` 层统一编排：

- 读取本地增量
- 推送远端变化
- 拉取远端变化
- 合并结果并回写本地状态
- 维护同步状态和错误提示

相关目录：

- `fastnote/src/processes/sync-notes/*`
- `fastnote/src/shared/api/pocketbase/*`

## 5. 实时能力

PocketBase realtime 用于接收远端笔记变化，并同步到本地状态。

相关实现：

- `fastnote/src/shared/api/pocketbase/realtime-service.ts`
- `fastnote/src/processes/session/*`

## 6. 前后端一体化宿主

后端使用 Go + PocketBase 作为宿主。

当前实现重点：

- 启动 PocketBase
- 注册 bootstrap
- 查找并挂载 `fastnote/dist`

相关实现：

- [main.go](/Users/yytest/Documents/projects/fast-note/backend/main.go)
- [static.go](/Users/yytest/Documents/projects/fast-note/backend/internal/server/bootstrap/static.go)

## 7. 当前实现边界

- 后端当前不包含业务 routes
- 后端当前不包含业务 schema
- 后端当前不包含业务 migrations
- 前端仍然是业务实现主体
