# 项目结构

本项目当前采用前后端一体仓库结构，前端目录为 `fastnote/`，后端目录为 `backend/`。

## 顶层结构

```text
backend/
  main.go
  internal/
    server/
      bootstrap/
      hooks/
  migrations/

fastnote/
  src/
  public/
  tests/
  src-tauri/
  package.json
  vite.config.ts

docs/
skills/
```

## 目录说明

### `backend/`

后端宿主目录，当前职责是运行 PocketBase 和托管前端静态资源。

- `main.go`：PocketBase 入口
- `internal/server/bootstrap/`：静态资源挂载
- `internal/server/hooks/`：后续服务端 hook 预留目录
- `migrations/`：后续 PocketBase schema 变更预留目录

说明：

- 当前阶段不包含业务 routes、业务表或业务 migrations

### `fastnote/`

前端目录，按 FSD 组织。

```text
fastnote/src/
  app/
  processes/
  pages/
  widgets/
  features/
  entities/
  shared/
```

各层职责：

- `app/`：应用启动、路由、全局装配
- `processes/`：同步、会话、公开笔记初始化、导航恢复
- `pages/`：路由页装配
- `widgets/`：业务 UI 模块
- `features/`：用户动作用例
- `entities/`：实体状态与规则
- `shared/`：工具、适配器、类型、通用 UI

### `docs/`

开发、产品和架构文档目录。

### `skills/`

项目内技能目录，用于约束 Codex/Copilot 类代理的工作方式。

## 关键原则

- 前端逻辑只放在 `fastnote/`
- 后端宿主逻辑只放在 `backend/`
- 前端通过 PocketBase SDK 访问后端
- 当前 `backend/` 是宿主层，不主动承载业务代码
