# 状态管理迁移记录

## 文档定位

本文记录笔记状态管理从早期集中式 store 继续拆分到当前 FSD 结构的过程。`src/stores/` 已不再承载运行时代码。

## 当前落点

- 私有笔记状态：`src/entities/note/model/state/note-store.ts`
- 公开笔记状态：`src/entities/public-note/model/state/public-note-store.ts`
- 笔记文件：`src/entities/note/model/use-note-files.ts`
- 同步编排：`src/processes/sync-notes`
- 页面消费：`src/pages/*`
- 业务列表与详情组合：`src/widgets/*`、`src/features/*`

## 历史迁移链路

```text
hooks/useNote.ts
  -> stores/notes.ts
  -> entities/note/model/state/note-store.ts

hooks/useUserPublicNotes.ts
  -> stores/publicNotes.ts
  -> entities/public-note/model/state/public-note-store.ts
```

## 当前原则

1. 领域状态放 `entities/*/model/state`
2. 跨页面编排放 `processes/*`
3. 用户动作放 `features/*`
4. 页面与 widget 只消费公开 API，不直接回到历史 store 目录

## 当前结论

这轮迁移已经从“重命名 store”推进到“按领域和流程拆分状态”。后续若要继续迁移，应优先减少页面对实体状态细节的直接了解，把更多编排逻辑压回 `features/processes`。
