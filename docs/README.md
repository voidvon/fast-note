# Fast-Note 文档导航

仓库中的所有文档统一收敛在 `docs/` 下，按用途分区：

## 核心说明

- `架构说明.md`：当前 FSD + DDD 架构主说明
- `部署说明.md`：发布构建、目录结构与运行方式说明
- `开发文档/`：技术栈、目录结构、核心模块、使用方式
- `产品文档/`：产品定位、功能说明、用户流程和迭代规划
- `产品原型/`：原型和交互说明
- `专题文档/`：同步机制、主题能力等专项说明

## 研发过程文档

- `需求文档/`：PRD、需求反推和需求补充
- `功能拆分与技术方案/`：模块级技术方案与总览
- `开发计划/`：迭代任务、组件契约、测试与发布计划

## 其他

- `页面/`：页面级说明
- `主题/`：主题相关说明
- `归档/`：历史版本或已被新文档替代的旧资料

如果是第一次接手项目，建议从下面顺序开始阅读：

1. `README.md`
2. `docs/架构说明.md`
3. `docs/开发文档/README.md`
4. `docs/产品文档/README.md`
5. 按需要进入 `docs/需求文档`、`docs/功能拆分与技术方案`、`docs/开发计划` 查看某次需求或迭代的过程文档
