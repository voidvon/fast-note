# 详情页富文本新建默认一级标题 PRD

## 1. 文档信息

- 文档标题：详情页富文本新建默认一级标题 PRD
- 版本：v0.1
- 日期：2026-03-11
- 作者：Codex
- 输入原文：
  - “新增一个需求，在详情页面 富文本中，希望在新建的时候，默认就是一级标题”
- 需求粒度：`功能级`

## 2. 原始需求摘要

- 原始事实：
  - 需求发生在详情页面的富文本编辑区域。
  - 场景是新建内容时。
  - 默认样式希望为一级标题。
- 目标问题：
  - 当前新建富文本时的默认文本样式不符合预期，用户希望开篇默认就是一级标题。
- 期望结果：
  - 用户进入新建详情页并开始输入时，首个输入位置默认使用一级标题格式。

## 3. 背景与问题陈述

- 当前背景：
  - 详情页存在富文本编辑能力。
- 要解决的问题：
  - 新建内容时，用户需要手动切换为一级标题，增加一步操作，且不符合预期写作起点。
- 本次目标：
  - 让新建场景下的富文本初始输入块默认采用一级标题。
- 非目标：
  - 不调整已有内容的历史样式。
  - 不扩展为所有段落默认一级标题。
  - 不涉及标题层级体系、样式主题或编辑器工具栏整体改版。

## 4. 用户与场景

- 用户角色：
  - 使用详情页进行笔记创建的普通用户。
- 核心场景：
  - 用户新建一条内容，进入详情页后直接开始输入标题性质的首行内容。
- 使用频率 / 时机：
  - `[待确认]`

## 5. 范围定义

### 5.1 In Scope

- I1：新建详情页首次进入富文本编辑态时，默认块类型为一级标题。
- I2：用户无需手动切换格式，直接输入即可得到一级标题样式。
- I3：仅作用于新建场景下的初始内容状态。

### 5.2 Out of Scope

- O1：编辑已有笔记时自动改写首行样式。
- O2：多级标题默认规则设计。
- O3：导出、同步、分享等下游能力改造。
- O4：富文本编辑器其他默认格式策略调整。

## 6. 功能需求

### FR-001 新建富文本默认一级标题

- 来源：`原始事实`
- 用户价值：
  - 用户创建新内容时可直接输入标题，减少一次格式切换操作。
- 前置条件：
  - 用户处于详情页面的新建场景。
  - 富文本内容尚未初始化为用户已有正文内容。
- 触发方式：
  - 用户新建一条内容并进入详情页面。
- 主流程：
  - 系统初始化富文本编辑器。
  - 首个可编辑输入块默认设置为一级标题。
  - 用户直接输入文本，展示为一级标题样式。
- 异常流程：
  - 若编辑器初始化失败，则回退为当前默认文本块类型，并记录日志。`[推断]`
- 业务规则：
  - 仅新建场景生效。
  - 仅首个初始输入块默认使用一级标题。
  - 用户后续可自行切换块类型，不应被再次强制改回一级标题。`[推断]`
- 数据变化：
  - 新建内容的首个块类型默认写入为 `H1 / 一级标题`。具体字段名 `[待确认]`
- 依赖与约束：
  - 依赖当前富文本编辑器支持块级默认类型配置。
  - 依赖新建和编辑已有内容场景可被明确区分。
- 待确认点：
  - 新建的判定标准是空白笔记、首次创建详情页，还是首次插入正文前。
  - 一级标题是否仅作为视觉样式，还是参与结构化数据存储。

### FR-002 既有内容不受影响

- 来源：`推断`
- 用户价值：
  - 避免修改历史内容或影响编辑已有笔记的习惯。
- 前置条件：
  - 用户打开的是已有内容详情页，或新建内容已存在用户输入。
- 触发方式：
  - 系统初始化编辑器时识别到非新建初始状态。
- 主流程：
  - 系统按现有内容结构渲染，不额外插入或替换为一级标题。
- 异常流程：
  - 若场景判定失败，可能导致已有内容被错误套用默认样式，应阻断该行为并保留原数据。`[推断]`
- 业务规则：
  - 已有首块内容类型保持原样。
  - 不对已保存数据执行迁移或批量修正。
- 数据变化：
  - 无新增数据写入，仅沿用现有内容。
- 依赖与约束：
  - 依赖编辑器初始化阶段可读取当前内容状态。
- 待确认点：
  - 草稿态未保存内容是否也应视为已有内容。

## 7. 非功能需求

- 性能：
  - 默认一级标题配置不应明显增加详情页编辑器初始化耗时。`[待确认]`
- 可用性：
  - 用户首次输入前应能直接看到一级标题输入态或对应样式反馈。
- 安全与权限：
  - 不涉及新增权限。`原始事实不足`
- 兼容性：
  - 需与当前详情页支持的平台保持一致。具体平台范围 `[待确认]`
- 可观测性：
  - 建议记录编辑器初始化类型，便于排查默认块类型未生效问题。`[推断]`
- 数据一致性：
  - 展示样式与实际保存的块类型需一致，避免显示为一级标题但存储为正文。

## 8. 验收标准

- AC-001（关联 FR-001）：
  - Given 用户新建一条内容并进入详情页
  - When 富文本编辑器完成初始化且用户开始输入首行
  - Then 首行内容默认以一级标题样式输入和展示

- AC-002（关联 FR-001）：
  - Given 用户在新建详情页尚未输入内容
  - When 用户未手动切换格式直接输入
  - Then 不需要额外操作即可生成一级标题块

- AC-003（关联 FR-002）：
  - Given 用户打开一条已有内容的详情页
  - When 页面进入编辑态
  - Then 系统保留原有块类型，不自动改为一级标题

- AC-004（关联 FR-002）：
  - Given 用户在新建场景首行已默认一级标题
  - When 用户手动切换为其他块类型继续编辑
  - Then 系统不应再次强制改回一级标题

## 9. 风险与未决问题

### 9.1 风险

- RISK-001：若新建与已有内容判定不准确，可能误伤已有笔记展示或编辑行为。
- RISK-002：若编辑器仅支持渲染态样式而未同步存储块类型，可能造成显示与数据不一致。
- RISK-003：若用户预期首行是标题、后续回车变正文，当前实现可能还需补充后续换行规则。`[推断]`

### 9.2 待确认问题

- Q-001：新建默认一级标题后，用户按回车创建下一行时，下一行应继续一级标题还是切回正文？
- Q-002：一级标题的默认行为是否只针对完全空白的新建内容？
- Q-003：该需求是否仅针对移动端详情页，还是所有端一致生效？
- Q-004：数据层中一级标题对应的具体富文本块类型或标记是什么？
