# Fast-Note 部署说明

本文档说明 Fast-Note 的发布构建产物、目录组成和运行方式，便于本地打包、交付和升级时核对。

## 源码与运行时组成

- `apps/web`：Web 前端源码、测试与 Vite 配置
- `apps/launcher`：Go 启动器，负责拉起官方 PocketBase 二进制
- `apps/desktop`：桌面端壳子，独立于服务端发布链路
- `pocketbase/pb_hooks`：PocketBase JavaScript hooks
- `pocketbase/pb_migrations`：PocketBase JavaScript migrations
- `pocketbase/version.json`：固定的 PocketBase 版本元数据

## 发布构建流程

```bash
npm install
npm run release
```

执行 `npm run release` 后，会依次完成以下步骤：

1. 构建 `apps/web`
2. 构建 `apps/launcher`
3. 下载当前平台对应的固定版本 PocketBase 二进制
4. 组装 `build/FastNote`
5. 生成当前平台更新归档与 `build/updates/latest.json`

## 发布目录结构

```text
build/FastNote/
  fastnote
  version.json
  backend/
    pocketbase
    pb_hooks/
    pb_migrations/
    pb_public/
  data/
  logs/
  README.md

build/updates/
  latest.json
  manifest-<version>-<platform>.json
  FastNote-<version>-<platform>.<ext>
```

## 运行说明

- 使用 `./fastnote` 启动服务
- 使用 `./fastnote version` 查看当前发布包版本元数据
- 使用 `FASTNOTE_UPDATE_MANIFEST_URL=/path/to/latest.json ./fastnote check-update` 检查更新清单
- 使用 `FASTNOTE_UPDATE_MANIFEST_URL=/path/to/latest.json ./fastnote update` 下载、校验并切换到新版本
- 使用 `FASTNOTE_UPDATE_MANIFEST_URL=/path/to/latest.json ./fastnote update --dry-run` 只完成下载、校验和 staging，不切换当前文件
- 使用 `./fastnote update-status` 查看当前运行态与最近一次更新结果
- 使用 `./fastnote cleanup-updates --keep 2` 清理旧的更新缓存目录
- 启动器内部会执行 `backend/pocketbase serve --dir ./data`
- `pb_public/` 目录由 PocketBase 直接作为静态资源目录提供
- 升级版本时应保留 `data/` 和 `logs/`，避免丢失数据和运行日志
- 当前 `update` 仅支持 macOS / Linux；Windows 平台尚未实现自替换 helper

## 部署检查项

- 确认目标机器具备当前平台可执行权限
- 确认 `build/FastNote/backend/` 下 PocketBase、hooks 和 migrations 已完整组装
- 确认首次启动后 `data/`、`logs/` 能正常创建和写入
- 如需覆盖升级，先备份现有 `data/` 目录
