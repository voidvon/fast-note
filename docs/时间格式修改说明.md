# 时间格式修改说明

## 修改概述

项目统一使用“日期与时间之间为空格”的时间字符串格式：

```text
2026-02-11 10:30:45.123Z
```

核心入口是 [index.ts](/Users/yytest/Documents/projects/fast-note/fastnote/src/shared/lib/date/index.ts) 中的 `getTime()`。

## 修改原因

部分数据库与过滤逻辑对空格分隔格式兼容性更好，同时保留 ISO 时间排序与 `Date` 解析能力。

## 当前主要承载文件

### 核心时间工具

1. `fastnote/src/shared/lib/date/index.ts`
   - 提供统一 `getTime()` 实现

2. `fastnote/src/shared/lib/storage/sync.ts`
   - 提供同步状态时间写入

### 远端与认证相关

3. `fastnote/src/shared/api/pocketbase/auth-service.ts`
   - 认证用户记录缺失时间时使用 fallback

### 领域与页面消费

4. `fastnote/src/entities/note/model/domain/note-rules.ts`
   - 删除保留窗口计算

5. `fastnote/src/pages/home/ui/home-page.vue`
   - 新建笔记与回收站过滤

6. `fastnote/src/pages/deleted/ui/deleted-page.vue`
   - 删除窗口过滤

7. `fastnote/src/features/note-actions-menu/ui/note-actions-menu.vue`
   - 立即永久删除时写入特殊时间戳

## 技术验证

### 兼容性测试

✅ 字符串比较顺序与时间顺序一致

```js
'2026-02-11 10:30:45.123Z' > '2026-02-11 09:30:45.123Z' // true
```

✅ JavaScript `Date` 仍可正常解析

```js
new Date('2026-02-11 10:30:45.123Z')
```

✅ 构建验证

```bash
npm run build
```

## 影响范围

### 不受影响的部分

- 所有通过 `getTime()` 获取当前时间的代码
- 大部分排序、过滤、同步比较逻辑
- IndexedDB 持久化与 PocketBase 同步

### 需要注意的部分

- 历史数据里可能仍保留带 `T` 的旧格式
- 新旧格式在运行期可以共存
- 若新增时间写入逻辑，应优先复用 `getTime()`，不要自行拼接格式

## 当前建议

1. 新增时间字段默认走 `fastnote/src/shared/lib/date/index.ts`
2. 同步流程中的当前时间写入统一走 `fastnote/src/shared/lib/storage/sync.ts`
3. 页面或 feature 不直接复制 `.replace('T', ' ')` 逻辑，优先复用公共工具
