# 现状实现与差距

## 1. 当前已经落地的能力

结合当前代码和既有方案文档，fastnote 的 AI 对话已经具备下面这些基础：

### 1.1 任务基础

- 已有 `AiAgentTask` 类型和步骤模型
- 已有任务状态、风险级别、确认模式、终止原因等字段
- 已有任务恢复相关快照字段

主要代码：

- `fastnote/src/features/ai-chat/model/agent-task.ts`
- `fastnote/src/features/ai-chat/model/use-ai-chat.ts`

### 1.2 上下文注入

- 已支持当前路由、当前活动笔记、当前目录、候选对象、最近对象注入
- 已支持从消息里解析 `/n/<noteId>` 和 `/f/...`
- 已支持“当前笔记 / 当前目录”这类自然语言指代

主要代码：

- `fastnote/src/features/ai-chat/model/request-context.ts`
- `fastnote/src/features/ai-chat/model/target-resolution.ts`
- `fastnote/src/features/ai-chat/model/route-target-snapshot.ts`

### 1.3 本地工具执行

- 已支持搜索、读取、列目录、创建、更新、移动、删除、加锁
- 已支持 dry-run / confirmed / requireConfirmation
- 已支持写操作预览和一部分风险确认

主要代码：

- `fastnote/src/features/ai-note-command/model/use-ai-note-command.ts`
- `fastnote/src/shared/types/ai-note-tools.ts`

### 1.4 连续工具调用

- 已支持模型返回 JSON envelope 请求工具
- 已支持执行工具后继续请求模型
- 已设置有限深度，避免无限续跑

主要代码：

- `fastnote/src/features/ai-chat/model/openai-compatible-chat-transport.ts`
- `fastnote/src/features/ai-chat/model/assistant-envelope.ts`
- `fastnote/src/features/ai-chat/model/use-ai-chat.ts`

### 1.5 风险控制与恢复

- 已支持高风险写操作确认
- 已支持待确认执行持久化
- 已支持一部分刷新恢复和路由保持

主要代码：

- `fastnote/src/features/ai-chat/model/mutation-policy.ts`
- `fastnote/src/processes/ai-chat-session/model/use-ai-chat-session.ts`

## 2. 当前实现的核心判断

当前 fastnote 已经不是纯聊天，也不只是“普通问答 + 一次工具调用”。

更准确的判断是：

- 它已经进入 Agent 首期实现阶段
- 但结构上仍偏“把多种能力堆在 `useAiChat()` 里”
- 还没有形成完整的 Agent 平台层

所以它现在是“可运行的首期任务型 Agent 原型”，不是“可持续扩展的 Agent 框架”。

## 3. 主要差距

### 3.1 缺少上下文压缩层

现状：

- 请求上下文是有的
- 但没有对长会话、长正文、长工具结果做统一压缩

影响：

- Prompt 容易膨胀
- 续跑输入不稳定
- 刷新恢复只能恢复状态，不能恢复任务摘要

### 3.2 缺少独立记忆层

现状：

- 有消息持久化
- 有待确认会话持久化
- 有当前任务持久化

但没有：

- 会话工作记忆
- 短期任务记忆列表
- 用户偏好记忆

### 3.3 缺少技能层

现状：

- 任务规则主要写在默认系统提示词中
- 没有技能注册和技能选择

影响：

- 规则维护成本高
- 很难按任务裁剪方法
- 难以让 Prompt 保持收敛

### 3.4 缺少应用内 MCP 协议层

现状：

- 已有工具类型和执行器

但没有：

- 方法注册表
- 输入输出 Schema 的统一入口
- 方法分组与白名单裁剪
- 技能到方法的绑定关系

## 4. 建议的近期演进顺序

为了不把系统一次性改散，建议按下面顺序推进：

1. 先补上下文压缩和任务摘要层。
2. 再补短期任务记忆和少量用户偏好记忆。
3. 再把现有工具整理成应用内 MCP 方法注册表。
4. 最后把高频场景抽成技能。

原因很简单：

- 没有压缩和记忆，工具越多越乱。
- 没有方法注册表，技能层会没有稳定落点。

## 5. 与现有文档的关系

当前 AI 相关文档里，已经覆盖但比较分散的部分包括：

- Agent 首期 PRD
- Agent 编排与状态机
- 对象定位与本地读取
- 写入确认与执行
- 恢复与路由保持
- 开发计划与组件契约

缺少、且本目录新增补上的部分主要是：

- “上下文压缩”专题说明
- “记忆”专题说明
- “技能与应用内 MCP”专题说明
- 统一入口和归档映射

## 6. 后续代码落点建议

如果继续做实现，建议新增下面几类文件：

- `fastnote/src/features/ai-chat/model/memory/*`
- `fastnote/src/features/ai-chat/model/skills/*`
- `fastnote/src/features/ai-chat/model/method-registry/*`

这样可以把现在集中在 `useAiChat()` 里的职责逐步拆开。
