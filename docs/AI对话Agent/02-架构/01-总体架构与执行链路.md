# 总体架构与执行链路

## 1. 设计原则

fastnote 的 AI 对话 Agent 需要坚持四个原则：

- local-first：本地状态和本地持久化优先，远端同步只做补偿。
- task-first：用户目标先变成任务，再驱动对话和方法执行。
- safe-write：所有写入经过风险分级与确认策略。
- web-contained：能力运行在网页端，因此扩展点优先设计为应用内协议，而不是依赖外部进程。

## 2. 总体分层

建议把 Agent 拆成 7 层：

### 2.1 交互层

负责用户输入、消息流、任务摘要、确认区、中断继续入口。

当前主要落点：

- `fastnote/src/features/ai-chat/ui/*`
- `fastnote/src/features/global-search/ui/global-search.vue`

### 2.2 编排层

负责把一次用户请求编排成任务，并驱动状态流转、方法续跑和结果收口。

当前主要落点：

- `fastnote/src/features/ai-chat/model/use-ai-chat.ts`
- `fastnote/src/features/ai-chat/model/agent-task.ts`

### 2.3 上下文层

负责采集当前路由、活动对象、候选对象、最近对象，并转成安全可控的请求上下文。

当前主要落点：

- `fastnote/src/features/ai-chat/model/request-context.ts`
- `fastnote/src/features/ai-chat/model/target-resolution.ts`
- `fastnote/src/features/ai-chat/model/route-target-snapshot.ts`

### 2.4 记忆层

负责短期任务记忆、对话压缩、长期偏好记忆和恢复快照。

当前状态：

- 仅有任务恢复和待确认恢复的一部分基础
- 没有独立记忆模型

### 2.5 技能层

负责把一类稳定任务抽成“技能模板”，约束 Prompt、可用方法、结果格式和风险边界。

当前状态：

- 未独立建层
- 主要能力仍堆在默认系统提示词里

### 2.6 应用内 MCP 层

负责以统一 Schema 暴露受控方法给 Agent 调用。

当前状态：

- 已有应用内方法原型
- 但还没有统一的方法注册协议和能力目录

### 2.7 执行层

负责真正调用笔记搜索、读取、写入、移动、删除、加锁和同步。

当前主要落点：

- `fastnote/src/features/ai-note-command/model/use-ai-note-command.ts`
- `fastnote/src/processes/ai-chat-session/model/use-ai-chat-session.ts`
- `fastnote/src/shared/types/ai-note-tools.ts`

## 3. 目标执行链路

完整链路建议如下：

1. 用户输入一条任务型请求。
2. 交互层创建会话消息并进入任务识别。
3. 编排层创建 `AiAgentTask`。
4. 上下文层收集当前页面上下文、对象快照和候选对象。
5. 记忆层决定是否注入压缩摘要、用户偏好、最近任务记忆。
6. 技能层根据意图选择技能模板和方法白名单。
7. 模型返回自然语言，或返回结构化方法调用请求。
8. 应用内 MCP 层校验方法名、参数、风险级别和确认要求。
9. 执行层执行本地方法，产出结构化结果。
10. 编排层决定是继续续跑、进入待确认、还是直接收口。
11. 任务结束后，把结果摘要写回短期记忆和恢复快照。

## 4. 任务状态机

建议沿用并收敛为下面这组状态：

- `identifying`
- `executing`
- `waiting_confirmation`
- `completed`
- `failed`
- `interrupted`
- `cancelled`

如果后续要补“读取中/规划中”，建议作为步骤类型，而不是继续扩展任务顶层状态，避免状态过碎。

## 5. 核心对象

### 5.1 任务对象

任务对象负责表达“这是一项执行任务”，而不是单条消息。

核心字段：

- `id`
- `input`
- `status`
- `steps`
- `riskLevel`
- `confirmationMode`
- `terminationReason`
- `routeTargetSnapshot`
- `lastResponseText`
- `lastRewriteSuggestion`

### 5.2 请求上下文

请求上下文负责表达“当前页面和当前对象提供了哪些可用线索”。

核心字段：

- `routePath`
- `activeNote`
- `activeFolder`
- `candidateNotes`
- `recentNotes`
- `resolvedTarget`

### 5.3 方法调用

方法调用负责表达“模型请求应用执行什么动作”。

核心字段：

- `methodName`
- `payload`
- `dryRun`
- `confirmed`
- `requireConfirmation`

### 5.4 方法结果

方法结果负责表达“执行后得到了什么结构化反馈”。

核心字段：

- `ok`
- `code`
- `message`
- `preview`
- `affectedNoteIds`
- `syncQueued`
- `humanActionRequired`

补充约束：

- 对外若使用 OpenAI `tool_calls` 或 Anthropic `tool_use`，它们只属于供应商协议层。
- 对内进入编排、压缩、恢复时，统一按 `method_call / method_result` 事务处理。

## 6. 当前实现映射

当前代码已经覆盖了总体链路中的一部分：

- 编排基础：`use-ai-chat.ts`
- 任务对象：`agent-task.ts`
- 上下文注入：`request-context.ts`
- URL/当前对象定位：`target-resolution.ts`
- 路由快照：`route-target-snapshot.ts`
- 风险策略：`mutation-policy.ts`
- 本地方法执行：`use-ai-note-command.ts`
- 待确认会话：`use-ai-chat-session.ts`

真正缺的是“压缩 / 记忆 / 技能 / 方法协议化”四层。

## 7. 下一步结构建议

后续如果要继续实现，建议在 `fastnote/src/features/ai-chat/model/` 周边新增三个子域：

- `memory/`
- `skills/`
- `method-registry/`

职责分别是：

- `memory/`：摘要压缩、短期记忆、长期记忆读取与写回
- `skills/`：技能声明、技能选择、技能约束
- `method-registry/`：应用内 MCP 方法定义、Schema 校验、风险分级、执行适配
