# AI 对话 Agent 文档

## 1. 目的

这个目录用于收口 fastnote 里与 AI 对话 Agent 相关的文档，统一回答下面几类问题：

- 它现在到底算不算 Agent。
- 现阶段已经实现了什么，缺什么。
- 下一步需要补哪些能力，分别落在哪一层。
- 旧 PRD、技术方案、开发计划分别对应哪一部分。

本目录采用“需求 / 架构 / 实现 / 归档”四层组织，后续关于 AI 对话 Agent 的新增结论，优先写到这里，再决定是否同步回其他专题目录。

## 3. 当前结论

当前 fastnote 的 AI 对话已经具备以下 Agent 基础能力：

- 任务状态机基础模型
- 当前页面与路由上下文注入
- 本地方法调用与写操作确认
- 有限深度的连续方法调用
- 刷新恢复与路由保持的一部分能力

但它还不是一套完整、可持续扩展的 Agent 体系，主要缺口集中在四个方向：

- 上下文压缩：缺少对长对话、长正文、多轮方法结果的统一裁剪策略。
- 记忆：缺少短期任务记忆、用户偏好记忆、可回收的长期记忆层。
- 技能：缺少面向任务模板的技能注册、选择、限制和输出约束。
- MCP：由于当前只在网页内使用，真正需要的不是外部 MCP Server，而是“应用内方法调用协议”。

## 5. 文档维护规则

- 需求边界、版本目标、验收口径，更新到“01-需求”。
- 架构分层、数据契约、执行链路，更新到“02-架构”。
- 当前代码状态、已落地能力、差距和迁移建议，更新到“03-实现”。
