# 任务状态与覆盖状态约定（fastnote）

## 1. 任务状态

| 状态   | 定义                     | 允许迁移到    |
| ------ | ------------------------ | ------------- |
| 未开始 | 已建任务，尚未开发       | 进行中 / 阻塞 |
| 进行中 | 正在开发或联调           | 已完成 / 阻塞 |
| 已完成 | 代码+测试+验证通过       | 无            |
| 阻塞   | 因依赖/决策/环境无法继续 | 进行中        |

## 2. 覆盖状态

| 状态   | 定义                       |
| ------ | -------------------------- |
| 未开始 | 用例未实现                 |
| 进行中 | 用例已编写但未稳定通过     |
| 已完成 | 用例稳定通过并接入计划门禁 |

## 3. 回写规则

- 任务状态改为 `已完成` 前，必须满足：
  - 关联代码已合并到当前交付变更
  - 关联自动化测试通过
  - 校验命令执行通过
- 若任务为 `P0`，覆盖状态不得为 `未开始`。
- 若出现阻塞，必须在任务行补充阻塞原因与负责人角色。
